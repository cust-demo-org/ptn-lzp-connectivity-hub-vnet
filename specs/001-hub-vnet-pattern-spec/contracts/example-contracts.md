# Example Contracts: PLZ Connectivity Hub VNet Pattern

**Date**: 2026-03-30 | **Branch**: `001-hub-vnet-pattern-spec`

This document defines the contracts for the AVM-style examples included with the pattern.

---

## Resource Module Decomposition Amendment

**Date**: 2026-04-09

### Changes to Examples

Examples must be rewritten to reflect the new decomposed architecture. The consumer-facing variable interface changes from a monolithic `hub_virtual_networks` map to individual resource maps.

- **`default/`**: Minimal deployment — 1 resource group, 1 VNet, 1 NSG. Demonstrates basic pattern usage with `network_security_group = { key }` cross-reference.
- **`full-dual-hub/`**: Full deployment — 1 resource group, 2 VNets (internet + intranet), 2 NSGs, 1 NAT gateway (with pattern-managed public IP via `public_ip_addresses.keys`), 5 public IPs (2 FW normal + 2 FW management + 1 NAT GW), 2 firewall policies (internet with DNS proxy + intranet), 2 firewalls (each with `public_ip_address.key` referencing pattern-managed PIPs). Demonstrates all key-based cross-references.
- Both examples use `terraform.tfvars` with `variables.tf` for passthrough.
- All example variables include a `description` attribute with a brief purpose statement and a pointer to the main module.

### Variable Mapping (Old → New)

| Old tfvars Key | New tfvars Key(s) |
|---------------|-------------------|
| `hub_virtual_networks.*.hub_virtual_network` | `virtual_networks.*` |
| `hub_virtual_networks.*.firewall` | `firewalls.*` |
| `hub_virtual_networks.*.firewall_policy` | `firewall_policies.*` |
| `hub_virtual_networks.*.bastion` | Bastion as Public IP + Bastion resource (TBD) |
| `hub_virtual_networks.*.virtual_network_gateways` | `virtual_network_gateways.*` |
| `hub_and_spoke_networks_settings` | DDoS via `virtual_networks.*.ddos_protection_plan` |
| `default_naming_convention` | Explicit names in each resource variable |

## Example Directory Structure Contract

All examples MUST follow AVM conventions:

```
examples/
├── .terraform-docs.yml          # Shared terraform-docs config
├── README.md                    # Boilerplate instructions
├── default/                     # REQUIRED — minimal viable deployment
│   ├── main.tf                  # terraform{} + provider{} + module call using ../../
│   ├── outputs.tf               # Outputs from the example
│   ├── _header.md               # Title + one-line description
│   ├── _footer.md               # Microsoft data collection notice
│   └── README.md                # Auto-generated by terraform-docs
└── full-dual-hub/               # Dual-hub: internet + intranet topology
    ├── main.tf
    ├── outputs.tf
    ├── _header.md
    ├── _footer.md
    └── README.md
```

## Example Conventions Contract

1. **No separate `terraform.tf`** — `terraform {}` block goes in `main.tf`
2. **Module source**: `source = "../../"` with commented registry hint
3. **Zero required input variables** — all values generated via random resources / naming modules
4. **Self-contained resource groups** — each example creates its own RG(s)
5. **README.md auto-generated** by `terraform-docs`
6. **Idempotent** — `terraform apply` twice = zero changes
7. **All AVM module parameters passthrough** — every parameter exposed by each module call in `main.tf` MUST originate from a root-level variable (passthrough from `terraform.tfvars`). No AVM module parameter may be hardcoded in `main.tf` unless it is a computed/derived value (resolved resource ID, merged tags, defaulted location)
8. **Example variable descriptions** — all passthrough variables in example `variables.tf` files MUST include a `description` attribute with a brief purpose statement and a reference to the main pattern module for complete details

## Example: `default/` (Minimal)

**Purpose**: Demonstrate the simplest possible single-hub deployment.

**What it exercises**:
- One resource group
- One hub VNet with firewall + bastion enabled
- One NSG associated with a user subnet
- LAW created by the wrapper (30-day retention)
- Core pattern feature flags at defaults
- Central `location`, `tags`, `enable_telemetry`

**What it does NOT exercise** (reserved for `full-dual-hub/`):
- Dual-hub topology (internet + intranet)
- NAT Gateways
- VPN/ExpressRoute gateways
- Hub-to-spoke peering (common services VNet)
- Flow logs / Network Watcher
- Storage accounts
- BYO resources

**`main.tf` contract**:
```hcl
terraform {
  required_version = ">= 1.13, < 2.0"
  required_providers {
    azurerm = { source = "hashicorp/azurerm", version = "~> 4.0" }
    random  = { source = "hashicorp/random",  version = "~> 3.0" }
  }
}

provider "azurerm" { features {} }

# Random region + naming
module "regions" { ... }
module "naming" { ... }

# Module call — minimal config
module "hub" {
  source = "../../"

  location         = <random_region>  # e.g. southeastasia
  enable_telemetry = false  # Disabled in examples
  tags             = { example = "default" }

  resource_groups = {
    rg_hub = { name = <unique_name> }
  }

  hub_virtual_networks = {
    hub_sea = {
      location = <random_region>  # e.g. southeastasia
      hub_virtual_network = {
        address_space = ["10.0.0.0/16"]
        subnets = {
          default = {
            name             = "snet-default"
            address_prefixes = ["10.0.1.0/24"]
            network_security_group = { resource_id = <resolved_nsg_id> }
          }
        }
      }
      firewall = { subnet_address_prefix = "10.0.0.0/26" }
      bastion  = { subnet_address_prefix = "10.0.0.64/26" }
    }
  }

  network_security_groups = {
    nsg_default = {
      name               = <unique_name>
      resource_group_key = "rg_hub"
    }
  }

  log_analytics_workspace_configuration = {
    name               = <unique_name>
    resource_group_key = "rg_hub"
  }
}
```

## Example: `full-dual-hub/` (Dual-Hub Internet + Intranet)

**Purpose**: Demonstrate a dual-hub topology matching the reference architecture — an internet egress/ingress hub and an intranet ingress hub, both peered to a common services VNet.

**Reference Architecture**:
```
┌────────────────────────────────────────────────────────┐
│  Common Services VNET (GEN Non Routable)               │
│  ┌──────────────────────┐                              │
│  │  DNS Resolver Subnet  │                              │
│  │  [DNS Resolver]       │                              │
│  └──────────────────────┘                              │
└────────┬────────────────────────────────────┬──────────┘
         │  Peering                           │  Peering
┌────────┴───────────────────┐  ┌─────────────┴──────────────┐
│ Internet Egress/Ingress    │  │ Intranet Ingress           │
│ VNET (Non Routable)        │  │ VNET (Routable)            │
│ ┌─────────────────────────┐│  │ ┌──────────────────────────┐│
│ │ [NAT GW] [Firewall]     ││  │ │ [Firewall]               ││
│ │ Firewall Subnet         ││  │ │ Firewall Subnet          ││
│ └─────────────────────────┘│  │ └──────────────────────────┘│
└────────────────────────────┘  └─────────────────────────────┘
```

**What it exercises**:
- Two resource groups (one connectivity, one common services)
- Two hub VNets in the same region (`southeastasia`):
  - **Internet Egress/Ingress** (`hub_internet`): Firewall (Standard) + NAT Gateway + Bastion, GEN non-routable address space
  - **Intranet Ingress** (`hub_intranet`): Firewall (Standard) only, GEN routable address space
- Common Services VNet with DNS Resolver subnet, peered to both hubs
- Mesh peering between the two hubs (core pattern's built-in mesh)
- NSGs on user subnets across both hubs
- NAT Gateway on the internet hub only
- Flow logs with wrapper-created storage account + traffic analytics
- Network Watcher configuration
- LAW created by wrapper
- **All parameters passthrough from variables** — no hardcoded values in module blocks

**`main.tf` contract** (high-level):
```hcl
# Two hubs in the same region — internet + intranet
hub_virtual_networks = {
  hub_internet = {
    location = "southeastasia"
    hub_virtual_network = {
      address_space = ["10.0.0.0/16"]  # GEN Non Routable
      subnets = {
        snet_workload = {
          name             = "snet-workload"
          address_prefixes = ["10.0.1.0/24"]
          network_security_group = { resource_id = null }  # resolved from nsg_key
          nat_gateway            = { resource_id = null }  # resolved from nat_gateway_key
        }
      }
    }
    firewall = {
      sku_tier              = "Standard"
      subnet_address_prefix = "10.0.0.0/26"
      zones                 = ["1", "2", "3"]
    }
    bastion = {
      subnet_address_prefix = "10.0.0.64/26"
      sku                   = "Standard"
    }
    enabled_resources = {
      firewall                              = true
      bastion                               = true
      virtual_network_gateway_vpn           = false
      virtual_network_gateway_express_route = false
      private_dns_zones                     = false
      private_dns_resolver                  = false
    }
  }

  hub_intranet = {
    location = "southeastasia"
    hub_virtual_network = {
      address_space = ["10.1.0.0/16"]  # GEN Routable
      subnets = {
        snet_workload = {
          name             = "snet-workload"
          address_prefixes = ["10.1.1.0/24"]
          network_security_group = { resource_id = null }  # resolved from nsg_key
        }
      }
    }
    firewall = {
      sku_tier              = "Standard"
      subnet_address_prefix = "10.1.0.0/26"
      zones                 = ["1", "2", "3"]
    }
    enabled_resources = {
      firewall                              = true
      bastion                               = false
      virtual_network_gateway_vpn           = false
      virtual_network_gateway_express_route = false
      private_dns_zones                     = false
      private_dns_resolver                  = false
    }
  }
}

# Global NSG map — shared across hubs
network_security_groups = {
  nsg_internet = { ... }
  nsg_intranet = { ... }
}

# NAT Gateway — internet hub only
nat_gateways = {
  natgw_internet = { ... }
}

# Common Services VNet peered to both hubs (DNS Resolver)
spoke_virtual_networks = {
  common_services_internet = {
    spoke_vnet_resource_id = <common_services_vnet_id>
    hub_key                = "hub_internet"
  }
  common_services_intranet = {
    spoke_vnet_resource_id = <common_services_vnet_id>
    hub_key                = "hub_intranet"
  }
}

# Storage for flow logs
storage_accounts = {
  sa_flowlogs = { ... }
}

# Flow log configuration
flowlog_configuration = {
  flow_logs = {
    fl_internet = {
      virtual_network = { key = "hub_internet" }
      storage_account = { key = "sa_flowlogs" }
      traffic_analytics = { enabled = true }
    }
    fl_intranet = {
      virtual_network = { key = "hub_intranet" }
      storage_account = { key = "sa_flowlogs" }
      traffic_analytics = { enabled = true }
    }
  }
}
```
