# Example Contracts: PLZ Connectivity Hub VNet Pattern

**Date**: 2026-03-30 | **Branch**: `001-hub-vnet-pattern-spec`

This document defines the contracts for the AVM-style examples included with the pattern.

---

## Resource Module Decomposition Amendment

**Date**: 2026-04-09

### Changes to Examples

Examples must be rewritten to reflect the new decomposed architecture. The consumer-facing variable interface changes from a monolithic `hub_virtual_networks` map to individual resource maps.

- **`default/`**: Minimal deployment — 1 resource group, 1 VNet, 1 NSG. Demonstrates basic pattern usage with `network_security_group = { key }` cross-reference.
- **`full-dual-hub/`**: Full deployment — 1 resource group, 2 VNets (internet + intranet), 2 NSGs, 1 NAT gateway (with pattern-managed public IP via `public_ip_addresses.keys`), 5 public IPs (2 FW normal + 2 FW management + 1 NAT GW), 2 firewall policies (internet with DNS proxy + intranet), 2 firewalls (each with `public_ip_address.key` referencing pattern-managed PIPs), 1 DNS resolver (internet hub with inbound/outbound endpoints and forwarding ruleset linked to intranet VNet). External resources: 1 RG, 1 VNet + subnet (PEP), 1 storage account + blob PEP, 1 private DNS zone. Cross-references: VNet peering to external VNet, BYO DNS zone links, flow logs referencing external storage account.
- Both examples use `terraform.tfvars` with `variables.tf` for passthrough.
- All example variables include a `description` attribute with a brief purpose statement and a pointer to the main module.

### Variable Mapping (Old → New)

| Old tfvars Key | New tfvars Key(s) |
|---------------|-------------------|
| `hub_virtual_networks.*.hub_virtual_network` | `virtual_networks.*` |
| `hub_virtual_networks.*.firewall` | `firewalls.*` |
| `hub_virtual_networks.*.firewall_policy` | `firewall_policies.*` |
| `hub_virtual_networks.*.bastion` | Bastion as Public IP + Bastion resource (TBD) |
| `hub_virtual_networks.*.virtual_network_gateways` | `virtual_network_gateways.*` |
| `hub_and_spoke_networks_settings` | DDoS via `virtual_networks.*.ddos_protection_plan` |
| `default_naming_convention` | Explicit names in each resource variable |

## Simplification Amendment

**Date**: 2026-03-30

### Changes to Examples

- **`default/`**: Removed `log_analytics_workspace_configuration` from module call and tfvars. Example now deploys: 1 resource group, 1 hub VNet with firewall + bastion, 1 NSG.

## Example Directory Structure Contract

All examples MUST follow AVM conventions:

```
examples/
├── .terraform-docs.yml          # Shared terraform-docs config
├── README.md                    # Boilerplate instructions
├── default/                     # REQUIRED — minimal viable deployment
│   ├── main.tf                  # terraform{} + provider{} + module call using ../../
│   ├── outputs.tf               # Outputs from the example
│   ├── _header.md               # Title + one-line description
│   ├── _footer.md               # Microsoft data collection notice
│   └── README.md                # Auto-generated by terraform-docs
└── full-dual-hub/               # Dual-hub: internet + intranet topology
    ├── main.tf
    ├── outputs.tf
    ├── _header.md
    ├── _footer.md
    └── README.md
```

## Example Conventions Contract

1. **No separate `terraform.tf`** — `terraform {}` block goes in `main.tf`
2. **Module source**: `source = "../../"` with commented registry hint
3. **Zero required input variables** — all values generated via random resources / naming modules
4. **Self-contained resource groups** — each example creates its own RG(s)
5. **README.md auto-generated** by `terraform-docs`
6. **Idempotent** — `terraform apply` twice = zero changes
7. **All AVM module parameters passthrough** — every parameter exposed by each module call in `main.tf` MUST originate from a root-level variable (passthrough from `terraform.tfvars`). No AVM module parameter may be hardcoded in `main.tf` unless it is a computed/derived value (resolved resource ID, merged tags, defaulted location)
8. **Example variable descriptions** — all passthrough variables in example `variables.tf` files MUST include a `description` attribute with a brief purpose statement and a reference to the main pattern module for complete details

## Example: `default/` (Minimal)

**Purpose**: Demonstrate the simplest possible single-hub deployment.

**What it exercises**:
- One resource group
- One hub VNet with firewall + bastion enabled
- One NSG associated with a user subnet
- LAW created by the wrapper (30-day retention)
- Core pattern feature flags at defaults
- Central `location`, `tags`, `enable_telemetry`

**What it does NOT exercise** (reserved for `full-dual-hub/`):
- Dual-hub topology (internet + intranet)
- NAT Gateways
- VPN/ExpressRoute gateways
- Hub-to-spoke peering (common services VNet)
- Flow logs / Network Watcher
- Storage accounts
- BYO resources

**`main.tf` contract**:
```hcl
terraform {
  required_version = ">= 1.13, < 2.0"
  required_providers {
    azurerm = { source = "hashicorp/azurerm", version = "~> 4.0" }
    random  = { source = "hashicorp/random",  version = "~> 3.0" }
  }
}

provider "azurerm" { features {} }

# Random region + naming
module "regions" { ... }
module "naming" { ... }

# Module call — minimal config
module "hub" {
  source = "../../"

  location         = <random_region>  # e.g. southeastasia
  enable_telemetry = false  # Disabled in examples
  tags             = { example = "default" }

  resource_groups = {
    rg_hub = { name = <unique_name> }
  }

  hub_virtual_networks = {
    hub_sea = {
      location = <random_region>  # e.g. southeastasia
      hub_virtual_network = {
        address_space = ["10.0.0.0/16"]
        subnets = {
          default = {
            name             = "snet-default"
            address_prefixes = ["10.0.1.0/24"]
            network_security_group = { resource_id = <resolved_nsg_id> }
          }
        }
      }
      firewall = { subnet_address_prefix = "10.0.0.0/26" }
      bastion  = { subnet_address_prefix = "10.0.0.64/26" }
    }
  }

  network_security_groups = {
    nsg_default = {
      name               = <unique_name>
      resource_group_key = "rg_hub"
    }
  }

  log_analytics_workspace_configuration = {
    name               = <unique_name>
    resource_group_key = "rg_hub"
  }
}
```

## Example: `full-dual-hub/` (Dual-Hub Internet + Intranet)

**Purpose**: Demonstrate a dual-hub topology with external resources for flow log storage integrated via pattern cross-references (peerings, BYO DNS zone links, flowlog configuration).

**Reference Architecture**:
```
┌──────────────────────────────────────────────────────────────────────────┐
│                        External Resources                               │
│  ┌─────────────────────────────────────────────┐                        │
│  │  Flowlog VNet (10.10.0.0/24)                │                        │
│  │  ┌────────────────────┐                     │                        │
│  │  │  snet-pep          │                     │                        │
│  │  │  [PEP → Blob SA]   │                     │                        │
│  │  └────────────────────┘                     │                        │
│  └────────┬─────────────────────────┬──────────┘                        │
│           │  Peering                │  Peering                          │
│  ┌────────┴───────────────────┐  ┌──┴──────────────────────────────┐    │
│  │ Internet Egress/Ingress    │  │ Intranet Ingress                │    │
│  │ VNET (10.0.0.0/16)        │  │ VNET (10.1.0.0/16)              │    │
│  │ ┌─────────────────────────┐│  │ ┌──────────────────────────────┐│    │
│  │ │ [NAT GW] [Firewall]     ││  │ │ [Firewall]                   ││    │
│  │ │ Firewall Subnet         ││  │ │ Firewall Subnet              ││    │
│  │ └─────────────────────────┘│  │ └──────────────────────────────┘│    │
│  └────────────────────────────┘  └─────────────────────────────────┘    │
│                                                                         │
│  Private DNS Zone: privatelink.blob.core.windows.net                    │
│    → linked to flowlog VNet, internet VNet, intranet VNet               │
│                                                                         │
│  Flow Logs: internet VNet + intranet VNet → external storage account    │
└──────────────────────────────────────────────────────────────────────────┘
```

**What it exercises**:
- One pattern-managed resource group (connectivity)
- Two hub VNets in the same region (`southeastasia`):
  - **Internet Egress/Ingress** (`vnet_internet`): Firewall (Standard) + NAT Gateway, non-routable address space
  - **Intranet Ingress** (`vnet_intranet`): Firewall (Standard) only, routable address space
- NSGs on user subnets across both hubs
- NAT Gateway on the internet hub only
- External resources (outside pattern module):
  - 1 resource group for flowlog infrastructure
  - 1 VNet + 1 subnet for storage account private endpoint
  - 1 storage account for flow log storage with blob private endpoint
  - 1 private DNS zone (`privatelink.blob.core.windows.net`)
- Cross-reference integration:
  - VNet peering: both hub VNets ↔ flowlog VNet (with reverse peering)
  - BYO DNS zone links: blob DNS zone → both hub VNets (via `virtual_network.key`)
  - Flow log configuration: both hub VNets → external storage account (via `storage_account_id`)
- Dynamic cross-references built in `main.tf` locals (merging with `var` values)
- **All pattern parameters passthrough from variables** — dynamic parts constructed in locals

**`main.tf` contract** (high-level):
```hcl
# External resources — outside pattern module
resource "azurerm_resource_group" "flowlog" { ... }
resource "azurerm_virtual_network" "flowlog" { address_space = ["10.10.0.0/24"] }
resource "azurerm_subnet" "pep" { address_prefixes = ["10.10.0.0/26"] }
resource "azurerm_storage_account" "flowlog" { ... }
resource "azurerm_private_dns_zone" "blob" { name = "privatelink.blob.core.windows.net" }
resource "azurerm_private_dns_zone_virtual_network_link" "blob_flowlog_vnet" { ... }
resource "azurerm_private_endpoint" "blob" { subresource_names = ["blob"] }

# Dynamic cross-references in locals
locals {
  # Merge peerings to flowlog VNet into each hub VNet
  virtual_networks = {
    for k, v in var.virtual_networks : k => merge(v, {
      peerings = merge(coalesce(try(v.peerings, null), {}), {
        peer_to_flowlog = {
          name                               = "peer-${k}-to-flowlog"
          remote_virtual_network_resource_id = azurerm_virtual_network.flowlog.id
          create_reverse_peering             = true
          reverse_name                       = "peer-flowlog-to-${k}"
        }
      })
    })
  }

  # BYO DNS zones — blob DNS zone → both hub VNets
  byo_private_dns_zones = merge(var.byo_private_dns_zones, {
    blob = {
      private_dns_zone_id = azurerm_private_dns_zone.blob.id
      virtual_network_links = {
        link_blob_internet = {
          name = "link-blob-to-internet"
          virtual_network     = { key = "vnet_internet" }
        }
        link_blob_intranet = {
          name = "link-blob-to-intranet"
          virtual_network     = { key = "vnet_intranet" }
        }
      }
    }
  })

  # Flow logs → external storage account
  # The pattern includes a time_sleep after VNet creation to wait for the
  # auto-provisioned Network Watcher, enabling single-step deployment.
  flowlog_configuration = {
    location = var.location
    flow_logs = {
      fl_internet = {
        enabled            = true
        name               = "fl-hub-internet"
        virtual_network    = { key = "vnet_internet" }
        storage_account_id = azurerm_storage_account.flowlog.id
        retention_policy   = { enabled = true, days = 7 }
      }
      fl_intranet = {
        enabled            = true
        name               = "fl-hub-intranet"
        virtual_network    = { key = "vnet_intranet" }
        storage_account_id = azurerm_storage_account.flowlog.id
        retention_policy   = { enabled = true, days = 7 }
      }
    }
  }
}

# Pattern module — uses locals for dynamic cross-references
module "hub" {
  source               = "../../"
  virtual_networks     = local.virtual_networks
  byo_private_dns_zones           = local.byo_private_dns_zones
  flowlog_configuration      = local.flowlog_configuration
  # ... other vars passed through
}
```
